# 美股 SEC Note 年报季报提取工具

[English README](README_EN.md)

这是一个面向最终用户的 Windows 桌面工具，用于基于 SEC Financial Statement and Notes Data Sets 提取美股年报和季报结果，并输出标准化 Excel 表格。

工具默认以 `note` 数据为主数据源，用户不需要修改代码。正常使用时，只需要选择数据目录、股票池工作簿、可选映射表和输出目录即可。

## 项目简介

本项目提供一个桌面软件，包含两个独立功能：

- `年报提取`
- `季报提取`

两种模式都会输出主结果表；如果提供映射表，还会额外输出修改映射表；同时还会输出未匹配公司清单，方便复核。

## 数据来源说明

SEC 官方下载地址：

- https://www.sec.gov/data-research/sec-markets-data/financial-statement-notes-data-sets

根据 SEC 官方页面说明，这套数据集具有这些特点：

- 数据集提供财务报表正文及附注中的文本和详细数值信息。
- 数据来源于公司向 SEC 提交的、采用 XBRL 格式报送的财务报告附件。
- 相比只保留财务报表数值信息的 Financial Statement Data Sets，这套 Financial Statement and Notes Data Sets 包含更多披露明细。
- 数据按“原始申报内容”提供，页面说明其展示方式经过扁平化处理，便于跨公司、跨时间对比分析。
- 数据集中还包含一些附加字段，例如公司的 Standard Industrial Classification，便于后续分析使用。
- 当前数据按月更新；SEC 页面说明，2020 年 11 月之前这套数据按季度更新。
- SEC 同时提醒：这套数据用于辅助分析，不应替代原始申报文件；做投资判断前仍应查看完整披露文件。

如果你第一次使用本工具，建议先阅读 SEC 官方页面和其配套说明文档，再准备本地 `note` 数据目录。

## 功能概览

- 从 SEC `note` 数据中提取美股年报结果
- 从 SEC `note` 数据中提取美股季报结果
- 季报模式下，优先使用最新季报；如果该公司在 `note` 中的最新年报 `filed_date` 更晚，则自动改用年报
- 年报和季报都支持可选映射表
- 年报和季报都会输出未匹配公司表
- 本地原件 / HTML 补齐默认隐藏，仅作为测试功能使用

## 输出文件

### 年报提取

固定输出：

- `美股年报提取_源表.xlsx`
- `美股年报提取_未匹配公司.xlsx`

可选输出：

- `美股年报提取_修改映射.xlsx`

说明：

- 只有在提供“年报映射表”时，程序才会输出修改映射表。

### 季报提取

固定输出：

- `美股季报提取_源表.xlsx`
- `美股季报提取_未匹配公司.xlsx`

可选输出：

- `美股季报提取_修改映射.xlsx`

说明：

- 只有在提供“季报映射表”时，程序才会输出修改映射表。

## 界面字段说明

### 季报提取

- `季报 note 数据目录`
  季报主数据源。程序会先提取公司最新季报，再与 `note` 中该公司的最新年报比较 `filed_date`，选择更新的一份。

- `主股票池工作簿`
  主股票池文件，决定本次要处理哪些公司。

- `补充股票池工作簿`
  补充公司清单，不需要时可以留空。

- `季报映射表（可选）`
  用于把 `segment_axis` 映射成 `segment_type`。只有填写后，才会额外输出 `美股季报提取_修改映射.xlsx`。

- `输出目录`
  结果保存位置。至少会输出季报源表和未匹配公司表。

- `本地年报原件目录（测试）`
  默认隐藏。只有勾选“启用测试功能”后才显示，仅用于测试时从本地 HTML / 原件做补齐。

### 年报提取

- `年报 note 数据目录`
  年报主数据源。程序会直接从这里提取最新年报数据。

- `主股票池工作簿`
  主股票池文件，决定本次要处理哪些公司。

- `补充股票池工作簿`
  补充公司清单，不需要时可以留空。

- `年报映射表（可选）`
  用于把 `segment_axis` 映射成 `segment_type`。只有填写后，才会额外输出 `美股年报提取_修改映射.xlsx`。

- `输出目录`
  结果保存位置。至少会输出年报源表和未匹配公司表。

- `本地年报原件目录（测试）`
  默认隐藏。只有勾选“启用测试功能”后才显示，仅用于测试时从本地原件做补齐。

## 映射表说明

年报和季报分别使用各自的映射表模板，不建议混用。

映射规则：

- 不要修改前两行表头
- 实际填写规则为 `segment_axis -> segment_type`

前两行表头应保持为：

第一行：

- `原始XBRL维度名称`
- `primary_disclosure_dim/disclosure_dims_available/segment_axis`

第二行：

- `segment_axis`
- `segment_type`

模板文件：

- [templates/季报映射模板.xlsx](templates/季报映射模板.xlsx)
- [templates/年报映射模板.xlsx](templates/年报映射模板.xlsx)

程序界面中也提供了模板生成功能：

- `生成季报映射模板`
- `生成年报映射模板`

## 使用步骤

### 直接运行软件

可执行文件不直接存放在仓库中。

对外分发建议使用 GitHub Releases 页面中的发布包。

### 季报提取流程

1. 打开软件并选择 `季报提取`
2. 选择 `季报 note 数据目录`
3. 选择 `主股票池工作簿`
4. 如有需要，选择 `补充股票池工作簿`
5. 如需输出修改映射表，选择 `季报映射表（可选）`
6. 选择 `输出目录`
7. 点击 `开始运行`

### 年报提取流程

1. 打开软件并选择 `年报提取`
2. 选择 `年报 note 数据目录`
3. 选择 `主股票池工作簿`
4. 如有需要，选择 `补充股票池工作簿`
5. 如需输出修改映射表，选择 `年报映射表（可选）`
6. 选择 `输出目录`
7. 点击 `开始运行`
