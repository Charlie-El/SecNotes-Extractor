# 美股 SEC Note 年报季报提取工具

[English Guide](README_EN.md)

美股 SEC Note 年报季报提取工具是一个面向业务用户的 Windows 桌面工具，用于基于 SEC Financial Statement and Notes Data Sets 提取美股年报和季报结果，并输出标准化 Excel 表格。工具默认以本地 `note` 数据为主数据源，正常使用时不需要修改代码。

软件提供两个独立功能：

- `年报提取`
- `季报提取`

两种模式都会输出主结果表和未匹配公司表；如提供映射表，还会额外输出修改映射表，便于后续核对、整理和交付。

## 主要功能

- 从 SEC `note` 数据中提取美股年报结果。
- 从 SEC `note` 数据中提取美股季报结果。
- 季报模式优先使用最新季报；如果同一公司在 `note` 中的最新年报 `filed_date` 更晚，则自动改用年报数据。
- 支持年报映射表和季报映射表，按需输出修改映射结果。
- 支持输出未匹配公司清单，便于复核缺失公司。
- 本地原件 / HTML 补齐默认隐藏，仅作为测试功能保留。

## 数据来源说明

本工具适用于 SEC 官方发布的 Financial Statement and Notes Data Sets。

官方下载地址：

- https://www.sec.gov/data-research/sec-markets-data/financial-statement-notes-data-sets

根据 SEC 官方说明，这套数据集主要用于提供财务报表正文及附注中的结构化文本和数值信息，数据来源于公司向 SEC 提交的 XBRL 财务报告附件。与只保留财务报表数值信息的 Financial Statement Data Sets 相比，这套数据集包含更多附注披露明细，适合做结构化提取、横向比较和批量分析。当前数据按月更新，早期版本曾按季度更新。

本工具不会自动下载 SEC 数据。使用前请先在本地准备好需要处理的 `note` 数据目录。

## 快速使用

如果你使用打包后的 Windows 软件，直接运行发布包中的 exe 文件即可。

如果你从源码运行，可在项目根目录执行：

```powershell
python -m ccxd_us_reports_app.app
```

如需重新打包桌面软件，可在项目根目录运行：

```powershell
.\build.ps1
```

打包完成后，发布文件会生成在本地 `release\` 目录中。

## 图形界面操作

1. 打开软件。
2. 选择 `年报提取` 或 `季报提取`。
3. 选择 `note` 数据目录。
4. 选择主股票池工作簿。
5. 如有需要，选择补充股票池工作簿。
6. 如需输出修改映射表，选择对应映射表。
7. 选择输出目录。
8. 点击 `开始运行`。
9. 运行完成后，到输出目录查看结果文件。

界面默认只显示正式使用所需字段。只有在勾选“启用测试功能”后，才会显示本地原件 / HTML 补齐相关输入项。

## 输入项说明

### 季报提取

- `季报 note 数据目录`
  季报主数据源。程序会先提取该公司的最新季报，再与 `note` 中该公司的最新年报比较 `filed_date`，选择更新的一份。

- `主股票池工作簿`
  决定本次需要处理哪些公司。

- `补充股票池工作簿`
  用于补充主股票池之外的公司，不需要时可以留空。

- `季报映射表（可选）`
  用于把 `segment_axis` 映射成 `segment_type`。只有提供后，程序才会额外输出季报修改映射表。

- `输出目录`
  用于保存最终结果文件。

- `本地年报原件目录（测试）`
  默认隐藏，仅在测试功能开启后显示。用于 `note` 无法覆盖时尝试从本地原件补齐。

### 年报提取

- `年报 note 数据目录`
  年报主数据源。程序会直接从这里提取最新年报数据。

- `主股票池工作簿`
  决定本次需要处理哪些公司。

- `补充股票池工作簿`
  用于补充主股票池之外的公司，不需要时可以留空。

- `年报映射表（可选）`
  用于把 `segment_axis` 映射成 `segment_type`。只有提供后，程序才会额外输出年报修改映射表。

- `输出目录`
  用于保存最终结果文件。

- `本地年报原件目录（测试）`
  默认隐藏，仅在测试功能开启后显示。用于测试时从本地原件补齐。

## 输出文件

### 年报模式

固定输出：

- `美股年报提取_源表.xlsx`
- `美股年报提取_未匹配公司.xlsx`

可选输出：

- `美股年报提取_修改映射.xlsx`

### 季报模式

固定输出：

- `美股季报提取_源表.xlsx`
- `美股季报提取_未匹配公司.xlsx`

可选输出：

- `美股季报提取_修改映射.xlsx`

只有在提供映射表时，软件才会生成对应的修改映射表。

## 映射表说明

年报和季报分别使用各自的映射表模板，不建议混用。

模板文件位于：

- [templates/季报映射模板.xlsx](templates/季报映射模板.xlsx)
- [templates/年报映射模板.xlsx](templates/年报映射模板.xlsx)

程序界面中也提供了：

- `生成季报映射模板`
- `生成年报映射模板`

映射规则：

- 不要修改前两行表头。
- 实际填写规则为 `segment_axis -> segment_type`。

前两行表头应保持为：

第一行：

- `原始XBRL维度名称`
- `primary_disclosure_dim/disclosure_dims_available/segment_axis`

第二行：

- `segment_axis`
- `segment_type`

## 注意事项

- 本工具不会自动下载 SEC 数据。
- 使用前请先准备本地 `note` 数据和股票池工作簿。
- 年报与季报是两个独立功能，输入项不完全相同。
- 只有提供映射文件时，才会生成修改映射表。
- 本地原件补齐默认隐藏，仅建议在测试场景下使用。
- 工具输出结果仍建议结合原始披露文件进行人工复核。
